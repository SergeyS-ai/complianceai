# PRD — ComplianceAI: Платформа для комплаенс-специалистов

**Версия:** 1.0  
**Дата:** 25 июня 2024  
**Статус:** Draft

---

## 1. Обзор продукта

ComplianceAI — внутренняя платформа для комплаенс-специалистов финансовых организаций, автоматизирующая генерацию профессиональных заключений на основе вводных данных и загружаемых документов. Платформа формирует рекомендации, готовит тексты для регулятора и предоставляет руководству управленческий дашборд для отслеживания работы команды.

---

## 2. Цели продукта

- Сократить время подготовки комплаенс-заключений с нескольких часов до 2–5 минут
- Стандартизировать качество заключений по всей команде
- Обеспечить актуальную нормативную базу (законы, ВНДБ) прямо в интерфейсе
- Дать руководству прозрачность по нагрузке, качеству и полезности инструмента
- Минимизировать риск регуляторных ошибок за счёт AI-анализа

---

## 3. Целевая аудитория

| Роль | Потребность |
|---|---|
| Комплаенс-специалист | Быстрая генерация заключений, доступ к нормативной базе |
| Старший специалист / руководитель направления | Проверка и согласование заключений высокого риска |
| Руководитель комплаенс-службы | Управленческий дашборд, аналитика по команде |

---

## 4. Типы заключений

Платформа поддерживает три режима, выбираемых до генерации:

### 4.1 ОД/ФТ
Заключения по противодействию отмыванию доходов и финансированию терроризма.  
Нормативная база: Закон РК «О ПОД/ФТ» № 191-V, Правила НБ РК № 199, Рекомендации ФАТФ, Типологии КФМ РК, ВНДБ.

### 4.2 МЭС (валютный контроль)
Заключения по операциям в иностранной валюте и трансграничным переводам.  
Нормативная база: Закон РК «О валютном регулировании» № 57, Правила НБ РК № 68, форматы ЕБНК.

### 4.3 Комплексно
Объединённое заключение по блокам ОД/ФТ + МЭС + внутренний комплаенс. Итоговый статус операции формируется по совокупности всех блоков.

---

## 5. Основной функционал

### 5.1 Генератор заключений

**Входные данные (форма):**
- Тип операции (трансграничный перевод, открытие счёта ЮЛ, ценные бумаги, торговое финансирование, кредитная операция, факторинг)
- Сумма в KZT
- Юрисдикция (направление перевода)
- Категория клиента (ЮЛ-резидент, ЮЛ-нерезидент, ФЛ-резидент, ПЭП)
- Уровень риска: Низкий / Средний / Высокий / Критический
- Применимые регуляторные нормы (множественный выбор: ФАТФ, ЗПДФТ РК, НБ РК, OFAC, AML EU, SWIFT, Basel IV, ВНДБ)
- Описание операции и дополнительный контекст (свободный текст)

**Загрузка документов:**
- Поддерживаемые форматы: PDF (`.pdf`), Word (`.doc`, `.docx`)
- Ограничение: до 20 МБ на файл, множественная загрузка
- AI читает содержимое документов и учитывает их при формировании заключения
- Загруженные файлы отображаются списком с названием, размером и кнопкой удаления

**Выходные данные (три вкладки):**

| Вкладка | Содержимое |
|---|---|
| Заключение | Свободный текст, адаптированный к типу проверки и уровню риска |
| Рекомендации | Пронумерованный список действий специалиста |
| Для регулятора | Готовый текст уведомления / СПО для НБ РК или КФМ РК |

**Экспорт и отправка:**
- Экспорт в PDF
- Экспорт в Word
- Копирование в буфер обмена
- Отправка регулятору (интеграция с каналом НБ РК / КФМ)

**Оценка качества:**
- Thumbs up / thumbs down после генерации
- Оценки агрегируются в аналитику

### 5.2 Боковая панель законодательства

Постоянно видна справа в интерфейсе генератора. Содержит четыре раздела:

**ОД/ФТ:**
- Закон РК «О ПОД/ФТ» № 191-V
- Правила НБ РК по ПОД/ФТ (Постановление № 199)
- ВНДБ РК — адаптация Рекомендаций ФАТФ
- Типологии ОД — методическое руководство КФМ РК

**Валютное регулирование (МЭС):**
- Закон РК «О валютном регулировании» № 57
- Правила НБ РК по валютным операциям (Постановление № 68)
- МЭС-отчётность — форматы ЕБНК

**Комплексный комплаенс:**
- Стандарт ISO 37301
- Требования НБ РК к внутреннему контролю (Постановление № 51)
- Санкционные списки OFAC / UN / EU (обновляются ежедневно)

**ВНДБ — внутренние документы:**
- Политика по управлению рисками ОД/ФТ
- Процедура KYC/KYB

**Возможности панели:**
- Поиск по названию и ключевым словам
- Раскрытие каждого документа с кратким описанием: что регулирует, пороги, требования
- Статусы: Действует / Обновлено / Изменён / Live (для санкционных списков)

### 5.3 История заключений

- Таблица всех сформированных заключений
- Столбцы: номер, тип операции, тип проверки (ОД/ФТ / МЭС / Комплексно), специалист, статус, дата
- Фильтрация по типу проверки и статусу
- Поиск по номеру, типу, специалисту
- Статусы: Завершено, На рассмотрении, Срочно, Черновик

### 5.4 Шаблоны

Готовые пресеты для типовых операций с привязкой к типу проверки:
- Трансграничный перевод (ОД/ФТ)
- Открытие счёта ЮЛ (МЭС)
- ПЭП-верификация (Комплексно)
- Ценные бумаги (МЭС)

Каждый шаблон содержит предзаполненные параметры и статистику использования.

## 6. Управленческий дашборд

Доступен руководителю комплаенс-службы.

### 6.1 Сводные метрики

| Метрика | Описание |
|---|---|
| Заключений за месяц | Общее количество, динамика vs прошлый месяц |
| Среднее время генерации | В минутах, сравнение с ручной работой |
| Рейтинг качества AI | Средняя оценка на основе обратной связи (шкала 1–5) |
| Загружено документов | Общее количество PDF + Word за период |

### 6.2 Аналитика использования

- Распределение заключений по типам (ОД/ФТ / МЭС / Комплексно) — столбчатая диаграмма
- Нагрузка по специалистам — количество заключений на каждого сотрудника
- Активность по дням — спарклайн за текущий месяц, пиковые дни
- Распределение по уровням риска — доля низкого / среднего / высокого / критического

### 6.3 Полезность функционала

Агрегированные оценки пользователей по каждому модулю:
- Генератор заключений
- Загрузка документов (PDF/Word)
- Рекомендации
- Текст для регулятора
- Панель законодательства
- Шаблоны

### 6.4 Лента активности

Последние действия в системе в реальном времени:
- Создание заключения
- Эскалация на руководителя
- Отправка в НБ РК / КФМ
- Автоматическая блокировка критических операций

---

## 7. Аналитика рекомендаций

Отдельный раздел для оценки качества AI-движка:

- Точность рекомендаций — % совпадения с финальным решением специалиста
- Принято без изменений — % заключений, не требовавших правок
- Экономия времени — часов в месяц, освобождённых от ручной работы
- Причины редактирования — топ категорий правок (уточнение формулировок, добавление деталей, изменение уровня риска и др.)

## 9. Нефункциональные требования

### Безопасность
- Все загружаемые документы обрабатываются внутри защищённого периметра
- Доступ к дашборду и аналитике только для руководящих ролей
- Журналирование всех действий с временными метками

### Производительность
- Генерация заключения без документов: не более 5 секунд
- Генерация с документами (PDF/Word): не более 30 секунд
- Чтение панели законодательства: мгновенно (кэш)

### Доступность
- Веб-интерфейс, десктоп-приоритет
- Поддержка светлой и тёмной темы

| Метрика | Цель |
|---|---|
